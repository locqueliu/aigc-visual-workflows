# aigc-visual-workflows

[English Version](./README.md)

这个仓库就是我拿来收 AIGC 视觉工作流的地方。

因为这类东西如果只活在截图、节点截图、聊天记录和各种临时文件里，过段时间基本就跟失踪差不多了。明明当时跑通过，回头一看只剩“好像做过”，细节全断。所以我想把这些工作流重新收成一套更能回看、也更能复用的结构。

## 这里现在放的东西

- 一个用来浏览工作流分类的小页面
- 放在 `workflows/` 里的工作流 JSON
- 一个检查字段完整性的校验脚本
- 一套我自己用着比较顺手的工作流记录格式

## 现在主要整理的几类工作流

- 商品详情图
- lookbook 场景搭建
- 品牌分镜和故事板规划

## 我为什么会这样整理

说白了就是不想每次都重复考古。

我一般会尽量把一条工作流至少留住这几样：

- 它到底是干什么的
- 它吃什么输入
- 它背后用了哪些模型和节点
- 它出结果以后要过哪些检查
- 它属于哪一类输出

## 命令

```bash
npm run validate:workflows
```

## 仓库结构

- `workflows/` 工作流 JSON 定义
- `scripts/validate-workflows.mjs` 字段校验脚本
- `index.html`、`styles.css`、`app.js` 浏览工作流的小页面

## 后面还会继续补的东西

- 更多实际生产里用得到的工作流分支
- 能公开放出来的预览图
- 模型路由和后处理上的补充说明
- 让这些工作流以后查找和复用都更顺的整理方式
